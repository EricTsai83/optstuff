This guide walks you through getting your first optimized image from OptStuff.

## Prerequisites

- An OptStuff account ([sign up here](https://optstuff.vercel.app))
- Bun 1.0+ (to run the TypeScript URL signer example)

## 1. Create a Team and Project

After signing in, you'll be guided to create a **personal team**. Then create your first **project** from the dashboard — a default API key is generated automatically.

> See [Project Setup](/getting-started/project-setup) for detailed UI instructions.

Before making requests, go to your project's **Settings → Domain Security → Image Sources** and add the domains you want to optimize images from (e.g., `https://images.unsplash.com`). Without at least one allowed source domain, all image requests will be rejected in production.

## 2. Copy Your Keys

After project creation, a success screen displays your keys:

- **Public Key** (`pk_...`) — identifies requests in URLs
- **Secret Key** (`sk_...`) — signs URLs server-side

> **Copy the secret key now.** It is only shown once.

## 3. Sign Your First URL

Use the standalone TypeScript example at `examples/optstuff-url-signer` ([view on GitHub](https://github.com/EricTsai83/optstuff/tree/main/examples/optstuff-url-signer)):

```bash
git clone https://github.com/EricTsai83/optstuff.git
cd optstuff/examples/optstuff-url-signer
cp .env.example .env
# Fill in OPTSTUFF_BASE_URL, OPTSTUFF_PROJECT_SLUG, OPTSTUFF_PUBLIC_KEY, OPTSTUFF_SECRET_KEY
bun run sign:url
```

Optionally, customize output settings:

```bash
bun run sign:url -- --width 800 --format avif
```

This prints a signed OptStuff URL to your terminal.
If you want to understand how signing works internally (payload format, HMAC-SHA256, expiration), read [URL Signing](/guides/url-signing).

## 4. Open the URL

Paste the output URL in your browser. You should see the image transformed based on your options (default: 400px width + WebP).
