This page uses a standalone CLI tool to verify your project keys and configuration are working correctly. No framework setup is required.

If you want to integrate directly into your application instead, skip to the [Next.js Integration](/getting-started/integrate-nextjs) guide.

## Prerequisites

- An OptStuff account with a project already created ([Quick Start](/getting-started/quickstart))
- Bun 1.0+ installed

## 1. Clone and Configure

Use the standalone TypeScript example at `examples/optstuff-url-signer` ([view on GitHub](https://github.com/EricTsai83/optstuff/tree/HEAD/examples/optstuff-url-signer)):

```bash
git clone https://github.com/EricTsai83/optstuff.git
cd optstuff/examples/optstuff-url-signer
cp .env.example .env
```

Fill in your `.env` file:

```bash
OPTSTUFF_BASE_URL="https://your-optstuff-deployment.com"
OPTSTUFF_PROJECT_SLUG="your-project-slug"
OPTSTUFF_PUBLIC_KEY="pk_..."
OPTSTUFF_SECRET_KEY="sk_..."
```

## 2. Sign a URL

```bash
bun run sign:url
```

This prints a signed OptStuff URL to your terminal (default: 400px width + WebP).

Optionally, customize the output:

```bash
bun run sign:url -- --width 800 --format avif
```

## 3. Open the URL

Paste the output URL in your browser. You should see the image transformed based on your options.

If the image loads correctly, your keys and configuration are working.

## What's Happening

The CLI tool performs the same HMAC-SHA256 signing that your application will do in production:

1. Builds an operation string from your options (e.g., `w_800,f_avif`)
2. Constructs a payload: `{operations}/{imageUrl}`
3. Signs it with your secret key using HMAC-SHA256
4. Outputs the full URL with the signature attached

To understand the signing formula in depth, see [URL Signing](/guides/url-signing).
