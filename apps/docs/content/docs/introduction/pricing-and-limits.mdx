### Default Rate Limits (per API Key)

| Limit | Default Value |
|-------|---------------|
| Per-minute | 60 requests |
| Per-day | 10,000 requests |

Default per-key limits are applied automatically. The data model supports per-key values, but dashboard UI controls for custom tuning are not yet available in this repository. See [Rate Limiting](/guides/rate-limiting) for current behavior.

## What Happens When You Exceed Limits

### Rate Limit Exceeded

When a single API key exceeds its per-minute or per-day limit, subsequent requests receive a `429 Too Many Requests` response with a `Retry-After` header indicating when to retry.

```json
{
  "error": "Rate limit exceeded",
  "reason": "Too many requests per minute",
  "retryAfter": 12,
  "limit": 60
}
```

Other API keys on the same project are **not affected** — rate limits are enforced independently per key.

### Monthly Usage Exceeded

When your team's total monthly requests or bandwidth reaches the free-plan threshold:

- Existing cached responses continue to be served by CDNs
- Dashboard usage views show the threshold status for monitoring
- In this repository, the `/api/v1` request path does not currently apply a separate monthly hard-stop check
